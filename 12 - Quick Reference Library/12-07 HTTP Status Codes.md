# 12-07 HTTP Status Codes

> [!abstract] Overview
> A production-grade HTTP status codes reference sheet. This guide covers status code classes (1xx to 5xx) and details diagnostics and resolutions for common client-side and server-side web application errors.

---

## 1. What Is It? (Concept Explanation)
HTTP status codes are 3-digit return codes generated by web servers in response to a client browser request. These codes represent the processing state of the request. They are divided into 5 standard classes:
- **1xx (Informational):** Request received, continuing process.
- **2xx (Success):** Action successfully received, understood, and accepted.
- **3xx (Redirection):** Further action must be taken to complete the request.
- **4xx (Client Error):** The request contains bad syntax or cannot be fulfilled.
- **5xx (Server Error):** The server failed to fulfill an apparently valid request.

*Seedha simple shabdon mein: HTTP status codes internet browse karte waqt web servers aur user browser ke beech ki communication report hain. Jab screen par error (jaise 404 ya 503) aata hai, toh yeh status codes support engineers ko batate hain ki problem client side (jaise incorrect web address ya permissions lock) par hai ya server side (jaise database server crash ya service maintenance) par.*

## 2. Common HTTP Status Codes Diagnostic Table
Use this reference key to isolate browser loading issues:

| Status Code | Type | Standard Name | Meaning | Troubleshooting Focus |
|---|---|---|---|---|
| **`200`** | Success | OK | Request succeeded. The server returned web page data. | None. Page loaded successfully. |
| **`301`** | Redirect | Moved Permanently | The URL has been updated. Browser is redirected to new URL. | Check redirect URL mapping. |
| **`400`** | Client Error | Bad Request | The server cannot process request due to client syntax error. | Clear browser cookie cache. |
| **`401`** | Client Error | Unauthorized | User lacks credentials. Authentication is required. | Verify user login inputs. |
| **`403`** | Client Error | Forbidden | The server understands identity but refuses authorization. | Verify security group access permissions. |
| **`404`** | Client Error | Not Found | The server cannot locate the requested file or page URL. | Verify URL string syntax and path. |
| **`500`** | Server Error | Internal Server Error | Generic error. Server crashed during processing. | Inspect web application error logs. |
| **`502`** | Server Error | Bad Gateway | Upstream routing server returned an invalid response. | Verify server proxies and configurations. |
| **`503`** | Server Error | Service Unavailable | Server is overloaded or down for planned maintenance. | Check backend database services. |
| **`504`** | Server Error | Gateway Timeout | Proxy server timed out waiting for upstream server response. | Check network routing latency. |

## 3. SOP: Troubleshooting Client Web Access Errors
Follow these diagnostic steps when a user reports web application errors:

### 1. Resolve Bad Request (`400`) & Forbidden (`403`)
1. Open the browser (Chrome/Edge).
2. Press `Ctrl + Shift + Delete` to open the Clear Browsing Data dialog.
3. Check the options **Cookies and other site data** and **Cached images and files**. Click **Clear Data**.
4. If accessing a secure corporate portal triggers a `403` error, open active directory and check if the user is in the correct security group linked to the web portal.

### 2. Diagnose Server Gateway Errors (`502` / `504`)
1. Run a traceroute from the client PC to verify the gateway hops:
   ```cmd
   tracert appserver.corp.local
   ```
2. Check local proxy settings in Windows Settings > Network & Internet > Proxy. Enable "Automatically detect settings" if manual proxy configurations are out of sync.

## 4. Study & Interview Q&A
**Q1: What is the primary difference between a `401` and a `403` HTTP error code?**
A: `401 Unauthorized` means the client identity is not verified. The user must provide valid credentials (username and password) to log in. `403 Forbidden` means the client identity has been authenticated, but the user account does not have authorization permissions to access the requested directory path.

**Q2: How do you troubleshoot a user experiencing HTTP 503 errors on the corporate intranet portal?**
A: An HTTP `503 Service Unavailable` error indicates the backend web server is online but unable to process the request, typically due to server overload or database maintenance. I check if other users are affected, notify the web application administrators, and verify if the web application pool services are running on the server.

**Q3: Why does clearing browser cookies resolve HTTP `400` errors?**
A: Web servers limit the size of HTTP headers. Cookies can become corrupted or balloon in size over time. When a browser sends an oversized or malformed cookie header, the server rejects the request with an HTTP `400 Bad Request` error. Clearing browser cookies flushes the stale data, resolving the syntax error.
