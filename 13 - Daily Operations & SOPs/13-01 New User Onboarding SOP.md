# 13-01 New User Onboarding SOP

> [!abstract] Overview
> Standard Operating Procedure (SOP) for new user onboarding in an enterprise Active Directory and Microsoft 365 environment. This document outlines mandatory identity verification, account provisioning, licensing, group memberships, and endpoint deployment.

---

## 1. What Is It? (Concept Explanation)
New user onboarding is the structured technical workflow of provisioning digital identities, access rights, software licenses, and physical computer assets for a new employee.
*Seedha simple shabdon mein: Jab koi naya employee company join karta hai, toh use system par login karne ke liye user ID, email account, security permissions aur ek configured laptop chahiye hota hai. IT support team is process ko end-to-end execute karti hai taaki day-1 par employee bina kisi issue ke apna kaam shuru kar sake.*

## 2. Onboarding Workflow Checklist
Follow these steps to complete a user onboarding ticket:

- [ ] **Verify HR Ticket Approvals:** Ensure the ticket is generated by HR and has manager approvals for requested roles and applications.
- [ ] **Provision AD Account:**
  1. Open Active Directory Users and Computers (`dsa.msc`).
  2. Navigate to the user's department Organizational Unit (OU) (e.g., `corp.local\Users\Finance`).
  3. Create the user object with the corporate naming standard: Firstname.Lastname (e.g., `rohan.sharma`).
  4. Configure UPN: `rohan.sharma@company.com`.
  5. Set a complex temporary password and check **User must change password at next logon**.
- [ ] **Assign AD Security Groups:**
  1. Add the user to default domain groups (e.g., `All-Employees-SG`).
  2. Assign department shares (e.g., `SG-Finance-ReadWrite`) based on the approved ticket.
- [ ] **Provision M365 Cloud Accounts:**
  1. Verify AD Connect has synchronized the account to Entra ID (sync cycles run every 30 minutes, or force it via `Start-ADSyncSyncCycle -PolicyType Delta`).
  2. Open M365 Admin Portal, locate the user, and assign the approved license (e.g., Office 365 E3).
  3. Configure mailbox settings and verify the email address is active.
- [ ] **Deploy Client Hardware:**
  1. Retrieve a laptop (e.g., Dell Latitude 5440) from the build room inventory.
  2. Image the laptop with the corporate Windows OS build (see [[13-02 New PC Setup & Imaging SOP]]).
  3. Verify baseline software (Chrome, Teams, Outlook, Antivirus) is installed.
  4. Link the asset tag barcode to the user's account in the CMDB register.

## 3. Real-World Incident Tickets

### Ticket 1: Stynchronization Delay on Day-1 Onboarding
- **Incident ID:** INC110291
- **Priority:** P3 (User Setup)
- **Problem Statement:** "A new finance employee started today but cannot log on to Outlook. The desktop client displays a credential prompt error."
- **Diagnostics:**
  1. Checked local ADUC. Confirmed the user `sneha.patel@corp.local` exists and the account is not locked out.
  2. Logged into Microsoft Entra admin portal. Found the user account exists, but the Exchange Online mailbox was missing.
  3. Checked the AD Connect sync client logs. Found the account was synchronized 5 minutes ago, but the mailbox provisioning status in Exchange Online was still pending database replication.
- **Resolution:** Manually forced mailbox provisioning by modifying the user's license profile (unassigned and reassigned the E3 license to trigger database synchronization). Verified mailbox creation in Exchange Online Admin Center, and assisted the user with Outlook configuration.

### Ticket 2: Missing Group Policy Access on Onboarding
- **Incident ID:** INC110305
- **Priority:** P3
- **Problem Statement:** "New HR manager cannot access the shared HR drive (H:\) on their new laptop."
- **Diagnostics:**
  1. Had the user open CMD and run `gpresult /r` to verify active group policies.
  2. Checked security group memberships. Found the user was member of `SG-All-Employees` but was missing the `SG-HR-Share` security group.
  3. Opened ADUC, located the user, and added them to `SG-HR-Share`.
  4. Had the user run `gpupdate /force` in CMD. The drive still failed to map.
- **Resolution:** Explained to the user that AD group memberships require a token refresh. Instructed the user to sign out of Windows and log back in (refreshing their Kerberos ticket). Verified the H:\ drive mapped successfully upon logon.

## 4. Technical Reference Configuration
Use this PowerShell snippet to automate AD user creation for onboarding batches:
```powershell
# Create new AD User with standard properties
New-ADUser -Name "Rohan Sharma" `
           -GivenName "Rohan" `
           -Surname "Sharma" `
           -UserPrincipalName "rohan.sharma@company.com" `
           -SamAccountName "rohan.sharma" `
           -Path "OU=Finance,OU=Users,OU=Corp,DC=corp,DC=local" `
           -AccountPassword (Read-Host -AsSecureString "Enter Temp Password") `
           -ChangePasswordAtLogon $true `
           -Enabled $true
```

## 5. Common Mistakes & Best Practices
- **Stale Passwords:** Do not write down the temporary password on sticky notes. Always transmit the temporary password securely via phone call or encrypted chat directly to the user or their hiring manager.
- **VLAN Mappings:** Ensure laptops are staged on the dedicated Staging VLAN. Staging devices on the production Wi-Fi VLAN can exhaust IP pools and slow down client deployments due to wireless speed limitations.

## 6. Pro Tips (Senior Engineer Secrets)
> [!tip] From the Field
> When provisioning users in hybrid AD environments, always check the `mailNickname` attribute in local AD Attribute Editor. If this attribute is blank, Microsoft Entra ID may fail to provision the user's mailbox in the cloud, leaving the user with an active login but no corporate email access.

## 7. Interview Q&A Bank
**Q1: How do you handle a scenario where a new hire arrives on Day 1 but has no onboarding ticket in the system?**
A: I explain to the hiring manager that for security compliance, we require an approved HR ticket to provision digital identities and access rights. I instruct them to submit an emergency request to HR. In the meantime, I can check if a physical asset (laptop) is prepped in inventory, but I will not create active credentials or grant domain access until the HR request is officially approved and logged.

**Q2: What is the purpose of the "User must change password at next logon" setting?**
A: This setting ensures that the IT technician who sets up the account does not retain knowledge of the user's active password. It maintains security integrity, transferring credential ownership solely to the user from their first login.

**Q3: How do you force active directory synchronization with Entra ID?**
A: I open PowerShell on the AD Connect server and run `Start-ADSyncSyncCycle -PolicyType Delta`. This forces an immediate differential synchronization of modified AD objects to the cloud, bypassing the default 30-minute scheduler.
